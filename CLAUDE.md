# Оборона Січі

Порт працюючого прототипу `_prototype.html` (ванільний JS + Canvas 2D,
ізометричний екшн/tower defense) у проєкт на Vite + TypeScript.

## ГОЛОВНЕ ПРАВИЛО
Це ПОРТ, а не переписування. Поведінка гри має лишитись ІДЕНТИЧНОЮ.

ЗАБОРОНЕНО:
- змінювати будь-які числа балансу (HP, швидкості, ціни, тайминги)
- "покращувати" чи оптимізувати алгоритми
- рефакторити ігрову логіку
- додавати нові фічі
- міняти візуал

ДОЗВОЛЕНО:
- переносити код 1:1
- додавати типи TypeScript
- розбивати на модулі
- перейменовувати змінні для читабельності

Якщо бачиш у прототипі щось неоптимальне — НЕ чіпай. Скажи мені словами,
я запишу в бекЛог (docs/BACKLOG.md).

## Зафіксовані рішення по порту
Єдині дозволені відхилення від коду прототипу. Поведінка і візуал
лишаються ідентичними.

1. `s.fire -= 0.016` з drawCannon → перенести в updateStructures.
   Крок лишається 0.016 ЗА КАДР, НЕ dt — інакше зміниться візуал.
2. drawCossack не читає keysVec() — тільки стан (p.step, p.fx, p.fy).
   Прапорець walking, який зараз рахується з вводу, оновлюється
   в updatePlayer і зберігається в стані гравця.
3. updateHUD() викликається НЕ з update(), а з game loop у main.ts:
   після update(dt), перед render(). Порядок і частота ті самі.

Відомі баги прототипу (див. docs/BACKLOG.md: B1 вороги застрягають
у ресурсах, B2 селяни без пошуку шляху, B3 селянин без гарячої клавіші)
переносяться ТОЧНО. Не виправляти.

## КРИТИЧНО: Float64Array у computeFlow
dist у computeFlow — тільки Float64Array. Постав там коментар:
"// НЕ МІНЯТИ на Float32 — похибка округлення робить частину карти
недосяжною". Ніколи не міняй цей тип.

## Стек
Vite + TypeScript + Canvas 2D. Без фреймворків. Без ECS.
UI — звичайний DOM поверх канваса.
