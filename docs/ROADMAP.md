# Оборона Січі — Roadmap

**Старт:** прототип `ai_studio_code_4.html` (1340 рядків, один файл, працює)
**Ціль:** грабельний MVP у браузері, задеплоєний на GitHub Pages, лінк у CV

**Загальний бюджет:** ~85 год роботи · ~220 промптів до Claude Code · ~250 ШІ-генерацій графіки
**Терміни:** 8–9 тижнів при 10 год/тиждень · 3 тижні при повному зануренні

---

## Ф0 — Рефакторинг у проєкт ✅→🔧
**12 год · ~30 промптів**

Прототип працює, але це один HTML на 1340 рядків. Далі так не масштабується.

- [ ] Vite + TypeScript, структура `src/`
- [ ] Розбити на модулі: `core/` (loop, camera, iso), `sim/` (economy, pathfinding, waves), `entities/`, `render/`, `ui/`, `data/`
- [ ] Винести всі константи (ціни, HP, швидкості) у `data/config.ts` — **єдине джерело правди для балансу**
- [ ] Типи: `Entity`, `Structure`, `ResourceNode`, `GameState`
- [ ] ESLint + Prettier
- [ ] Репозиторій на GitHub, GitHub Pages через Actions (деплой одразу, не в кінці!)

**Ризик:** спокуса переписати все з нуля. Не треба. Портуй код, що працює, як є — рефактор поведінки нуль.

**Готово, коли:** `npm run dev` дає ту саму гру, що й прототип, але з 15 файлів замість одного.

## Ф1 — Економіка з зубами
**20 год · ~55 промптів**

Зараз економіка — це лічильник, що росте. Треба зробити її системою з тиском.

- [ ] **Сіль** як 4-й ресурс + солончаки на карті + солеварня
- [ ] **Хата** (+ліміт селян) — без неї селян не наймеш
- [ ] **Комора** (+ліміт складу) — стеля на ресурси
- [ ] **Upkeep:** селяни і козаки їдять зерно щохвилини. Нема зерна → селяни розбігаються
- [ ] **Ліміт селян**, лічильник у HUD: `👤 4/6`
- [ ] Ланцюжок: солончак → солеварня → сіль → чумак → 🪙
- [ ] UI: панель будівництва з категоріями (Економіка / Оборона), tooltips із цінами
- [ ] Селяни **тікають** від ворога до Січі замість тупо стояти й гинути

**Найважче тут:** баланс upkeep. Занадто жорсткий — гра нечесна. Занадто м'який — його ніхто не помітить. Закладай 4–5 год суто на підбір цифр.

## Ф3 — Арт-пайплайн
**18 год · ~40 промптів · ~250 генерацій**

Найдовша фаза за календарним часом, бо генерація — це ітерації.

### Крок 1: Style bible (2 год)
Згенеруй **хату** першою. Перебирай, поки не буде ідеально. Вона стає референсом до кінця проєкту.

Базовий шаблон промпту — міняєш лише перший рядок:

```
[ОБ'ЄКТ], isometric game sprite, single object centered on plain
white background, no cast shadow, 2:1 isometric projection viewed
from above at 30 degrees.

Style: hand-painted 2D game art, clean readable silhouette, muted
historical palette — wheat gold, weathered timber brown, clay ochre,
moss green, rusted iron. Slight painterly texture, soft ambient
light from upper left.

Setting: 17th century Ukrainian Cossack frontier settlement on the
wild steppe. Authentic period materials: wattle-and-daub khata,
thatched straw roof, whitewashed clay walls, rough-hewn oak logs.
No European stone castles, no fantasy elements, no modern objects.

No text, no UI, no characters, no border, no watermark.
```

Підстановки:
| Об'єкт | Перший рядок |
|---|---|
| Хата | `Ukrainian peasant khata, whitewashed clay walls, thatched roof` |
| Комора | `wooden granary on stilts, thatched roof, small ladder` |
| Солеварня | `salt evaporation pans, wooden frames, mounds of drying white salt` |
| Кузня | `village smithy, open forge, anvil, stone chimney` |
| Частокіл | `palisade wall segment, sharpened vertical oak logs` |
| Вежа | `wooden watchtower, log construction, thatched canopy` |
| Січ | `fortified Cossack church with wooden bell tower, palisade ring` |
| Ліс | `cluster of steppe oak trees, dense foliage` |
| Пшениця | `patch of ripe golden wheat` |

Іконки — окремий шаблон: прибрати `isometric`, додати `flat game icon, front view, thick outline`.

### Крок 2: Юніти (CC0, не ШІ)
ШІ не тримає консистентність між кадрами анімації. Не бийся — бери готове:
- **Kenney** (CC0, є дзеркала на GitHub) — top-down/iso персонажі
- **LPC / Universal LPC Spritesheet** (CC-BY-SA / GPL) — 4 напрямки, ходьба + атака
- **OpenGameArt.org**, тег `isometric` + `CC0`

Перефарбувати в палітру + додати шаблю/шаровари — це 1–2 год у будь-якому редакторі, і виглядатиме твоїм.

### Крок 3: Інтеграція (10 год)
- [ ] Скрипт: cutout фону, обрізка, ресайз → `assets/raw/` → `assets/atlas/`
- [ ] Збірка texture atlas (TexturePacker free або власний скрипт)
- [ ] Асет-лоадер із прелоадером
- [ ] Заміна всіх `ctx.beginPath()` малювалок на `drawImage`
- [ ] Y-сортування спрайтів (уже є в `items.sort` — просто підмінити рендер)

**Реалістика:** із 10 згенерованих картинок у гру піде 1–2. Тому 250 генерацій → ~30 фінальних асетів.

## Правила, щоб не злити проєкт

1. **Деплой на GitHub Pages у Ф0**, не в Ф4. Гра, яку не видно за лінком, не існує.
2. **Один коміт = одна працююча гра.** Ніколи не залишай main зламаним.
3. **Не чіпай backlog з GAME_DESIGN §9.** Взагалі. Навіть якщо дуже хочеться.
4. **Грай у власну гру щотижня** по 10 хвилин. Це єдиний спосіб помітити, що вона нудна.
5. Якщо фаза перевищує естімейт на 50% — **ріж скоуп фази**, не переношуй дедлайн.
6. Дай грати комусь третьому після Ф2. Не після Ф4 — тоді буде пізно щось міняти.
